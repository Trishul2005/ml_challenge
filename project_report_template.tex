\subsection{Choosing a Model}

\subsubsection{Exploration}
When considering which model to use, we considered three main candidates: Na\"ive bayes, multi-layer perceptrons, and random forests. To evaluate which model to choose we looked at the three models average performance as well as peak performance over hundreds of different training-test splits. We did this because we noticed that there is a significant difference in validation accuracy depending on our split on untuned models, so evaluating on one split could bias one model over another. We kept evaluated the same split for each of the three models and found that random forests had the highest validation accuracy on avg and in its extrema before hyperparameter tuning. We also did a higher volume of random splits on each model without keeping them consistent as that was faster when we considering a single model and the results were consistent with when we homologated the splits between model families.
This gave us confidence that the random forest family would provide the best model we could train as it outpreformed Na\"ive bayes and MLPs in both mean and peak validation accuracy across all runs.

Once we went with Random forest, we considered 2 hyperparameters to tune: tree depth and number of trees. We went this these because they determine how tightly each tree can approximate a function and how many such trees we would be averaging across. We did binary search on two axes, in each iteration we took one hyperparameter and shifted it, leaving the other constant. This was to reduce ambiguity of the exact cause of decrease/increase of performance. We evaluated performance by a model's mean validation error across many splits. Once again since split choice was a large source of variance we ran each test 10-20 times across different splits allowing the random forest to be trained 100 times per choice of split to allow for variance due to ties to average out. 

Specifically consider this table with a selection of parameter pair and metrics we got for them:
